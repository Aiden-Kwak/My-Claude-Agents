# /retrospect — 프로젝트 회고 및 교훈 기록

프로젝트 완료 후 또는 작업 중 발생한 문제를 분석하여 `CLAUDE.md`의 `## 누적 교훈`에 기록합니다.

---

## 실행 순서

### 1. 현재 상태 파악

다음 파일들을 읽어 이번 작업을 파악하세요:

```
Read("CLAUDE.md")               # 기존 교훈 확인 (중복 방지)
Read("docs/qa-report.md")       # QA에서 발견된 이슈
Read("docs/bug-report.md")      # 버그 목록
```

### 2. 분석 항목

다음 질문에 답하여 교훈을 도출하세요:

**설계 단계**
- requirements.md가 명확했나? 불명확한 부분이 있었나?
- 설계 결정 중 나중에 문제가 된 것이 있었나?

**구현 단계**
- 예상치 못한 기술적 문제가 있었나?
- 환경 설정(DB, Docker, 패키지)에서 막힌 부분이 있었나?
- 에이전트가 반복적으로 같은 실수를 했나?

**QA 단계**
- 테스트 작성/실행에서 어려움이 있었나?
- 빌드 실패 원인은 무엇이었나?
- 코드 리뷰에서 반복 패턴이 있었나?

**프로세스**
- 어느 단계에서 가장 많은 시간이 걸렸나?
- 병렬 실행이 효과적이었나?
- start.sh가 제대로 동작했나?

### 3. CLAUDE.md 업데이트

`CLAUDE.md`의 `## 누적 교훈` 섹션에 다음 형식으로 추가:

```markdown
### [YYYY-MM-DD] | [프로젝트명]
**에이전트**: [해당 에이전트 또는 "전체"]
**문제**: [발생한 문제 또는 비효율]
**해결**: [어떻게 해결했는지]
**교훈**: [다음 프로젝트에서 반드시 기억할 핵심 내용]
```

교훈이 여러 개면 각각 별도 항목으로 작성하세요.

### 4. 개선 제안 (선택)

CLAUDE.md 또는 에이전트 파일에 개선이 필요하면 직접 수정하세요:
- 코딩 규칙 추가/수정
- 에이전트 작업 순서 개선
- 자주 발생하는 문제에 대한 사전 방지책 추가

## 예시 교훈 기록

```markdown
### 2026-02-25 | Todo CRUD 서비스

**에이전트**: db-agent
**문제**: settings.py의 parse_database_url이 postgresql:// 스킴만 처리하여 테스트 실행 불가
**해결**: config/test_settings.py를 생성하여 SQLite in-memory로 오버라이드
**교훈**: 백엔드 구현 시 항상 config/test_settings.py (SQLite)를 함께 생성할 것

---

**에이전트**: start.sh
**문제**: macOS에서 Colima 데몬이 꺼져 있어 ./start.sh 실행 시 Docker 오류 발생
**해결**: start.sh에 `colima start` 자동 실행 로직 추가
**교훈**: start.sh는 항상 Docker/Colima 상태 확인 및 자동 시작 로직을 포함할 것
```
